    Regulatory Compliance Detectors

    SlowQL provides comprehensive regulatory compliance detection for GDPR, HIPAA, and PCI-DSS requirements. This guide covers all 41 compliance patterns implemented in Sprint 1.

    üìã Overview

    Regulatory compliance is critical for organizations handling sensitive data. SlowQL's Compliance Detector Suite provides:

    • 41 comprehensive compliance patterns covering GDPR, HIPAA, and PCI-DSS requirements
    • GDPR Data Protection (20 patterns) - EU data protection and privacy rights
    • HIPAA Healthcare (10 patterns) - Protected Health Information security
    • PCI-DSS Payment (5 patterns) - Credit card and payment data protection
    • Real-world compliance scenarios based on regulatory requirements
    • Audit-ready reporting with regulatory references and remediation guidance

    üá™üá∫ GDPR Compliance Patterns (20 patterns)

    The General Data Protection Regulation (GDPR) requires strict controls over personal data processing. These detectors identify potential violations.

    Data Subject Rights (Articles 12-22)

    Data Subject Rights Violation

    -- Detected patterns:
UPDATE user_account WHERE id=123 WITHOUT user_consent
DELETE FROM profile WHERE user_id=456 WITHOUT data_subject_request

    Rule: COMP-GDPR-002 - Data Subject Rights Violation Impact: Violates GDPR Articles 16-17 (rectification and erasure) Fix: Implement proper data subject request procedures

    Access Request Prevention

    -- Detected patterns:
SELECT personal_data FROM users WHERE export=false
SELECT user_info FROM accounts WHERE access_request=restricted

    Rule: COMP-GDPR-015 - Data Access Request Prevention Impact: Violates GDPR Article 15 (right of access) Fix: Implement comprehensive data discovery and export mechanisms

    Rectification Prevention

    -- Detected patterns:
UPDATE personal_info SET name='fixed' WHERE immutable=true
SELECT * FROM user_data WHERE read_only=true

    Rule: COMP-GDPR-014 - Data Rectification Prevention Impact: Violates GDPR Article 16 (right to rectification) Fix: Implement data rectification mechanisms with audit trails

    Data Portability Prevention

    -- Detected patterns:
SELECT user_data FROM database WHERE export=restricted
SELECT personal_info FROM users WHERE download=blocked

    Rule: COMP-GDPR-013 - Data Portability Prevention Impact: Violates GDPR Article 20 (right to data portability) Fix: Implement data export in structured formats (CSV, JSON)

    Consent and Lawfulness (Articles 6-7)

    Missing Consent Tracking

    -- Detected patterns:
PROCESS personal_data WITHOUT consent_tracking
COLLECT user_information missing consent mechanisms

    Rule: COMP-GDPR-003 - Missing Consent Tracking Impact: Violates GDPR Article 7 (conditions for consent) Fix: Implement consent tracking with timestamps and withdrawal mechanisms

    Consent Withdrawal Prevention

    -- Detected patterns:
UPDATE user_account WHERE consent=true WITHOUT option_to_withdraw
DELETE FROM preferences WHERE user_consent=false

    Rule: COMP-GDPR-009 - Consent Withdrawal Prevention Impact: Violates GDPR Article 7(3) (withdrawal of consent) Fix: Implement easy-to-use consent withdrawal mechanisms

    Missing Legal Basis

    -- Detected patterns:
PROCESS personal_data WITHOUT legal_basis
ANALYZE user_information unclear purpose

    Rule: COMP-GDPR-011 - Missing Legal Basis Impact: Violates GDPR Article 6 (lawfulness of processing) Fix: Document legal basis (consent, contract, legal obligation, etc.)

    Data Minimization and Purpose (Article 5)

    Excessive Data Collection

    -- Detected patterns:
SELECT ALL personal_data FROM users
SELECT * FROM customer_information unnecessary_fields

    Rule: COMP-GDPR-004 - Excessive Data Collection Impact: Violates GDPR Article 5(1)(c) (data minimization) Fix: Collect only necessary data for stated purposes

    Purpose Limitation Violation

    -- Detected patterns:
USE marketing_data originally_collected_for service_provision
PROCESS advertising_information beyond original purpose

    Rule: COMP-GDPR-010 - Purpose Limitation Violation Impact: Violates GDPR Article 5(1)(b) (purpose limitation) Fix: Obtain additional consent for secondary uses

    Lack of Transparency

    -- Detected patterns:
PROCESS user_data INVISIBLE algorithm logic
ANALYZE personal_information UNDOCUMENTED purpose

    Rule: COMP-GDPR-012 - Data Processing Transparency Impact: Violates GDPR Articles 13-14 (information to be provided) Fix: Provide clear information about processing purposes and logic

    Right to Erasure (Article 17)

    Right to Erasure Prevention

    -- Detected patterns:
CREATE TABLE user_data FOREIGN KEY REFERENCES user_account ON DELETE RESTRICT
ALTER TABLE personal_info CONSTRAINT references user_data ON DELETE CASCADE

    Rule: COMP-GDPR-005 - Right to Erasure Prevention Impact: Violates GDPR Article 17 (right to be forgotten) Fix: Implement cascading deletes or soft deletes with audit trails

    Cross-border Transfers (Chapter V)

    Cross-border Transfer Risk

    -- Detected patterns:
CONNECT server US database eu_to_usa_data
SELECT data FROM host CANADA without safeguards

    Rule: COMP-GDPR-006 - Cross-Border Transfer Risk Impact: Violates GDPR Chapter V (transfers to third countries) Fix: Implement Standard Contractual Clauses (SCCs)

    International Transfer Safeguards

    -- Detected patterns:
TRANSFER personal_data OUTSIDE EU without_safeguards
STORE user_information THIRD_COUNTRY no adequacy decision

    Rule: COMP-GDPR-019 - International Transfer Safeguards Impact: Violates GDPR Chapter V (transfers to third countries) Fix: Implement approved transfer mechanisms or adequacy decisions

    Profiling and Automated Decisions (Article 22)

    Automated Profiling Detection

    -- Detected patterns:
IMPLEMENT automated decision user CLASSIFICATION without safeguards
USE algorithm MACHINE_LEARNING scoring customer profile

    Rule: COMP-GDPR-007 - Automated Profiling Detection Impact: Violates GDPR Article 22 (automated decision-making) Fix: Implement human review mechanisms and provide explanations

    Data Retention (Article 5(1)(e))

    Excessive Data Retention

    -- Detected patterns:
RETAIN personal_data INDEFINITELY
KEEP user_information FOREVER permanent storage

    Rule: COMP-GDPR-008 - Excessive Data Retention Impact: Violates GDPR Article 5(1)(e) (storage limitation) Fix: Implement data retention policies with automatic deletion

    Breach Notification (Articles 33-34)

    Breach Notification Requirements

    -- Detected patterns:
UNAUTHORIZED access personal_data unusual activity
SUSPICIOUS export sensitive_information without proper logging

    Rule: COMP-GDPR-016 - Breach Notification Requirements Impact: Violates GDPR Articles 33-34 (breach notification) Fix: Implement breach detection and notification within 72 hours

    High-risk Processing (Article 35)

    DPIA Requirement

    -- Detected patterns:
LARGE_SCALE systematic MONITORING special categories data
AUTOMATED processing SENSITIVE information high risk

    Rule: COMP-GDPR-017 - DPIA Requirement Impact: Violates GDPR Article 35 (data protection impact assessment) Fix: Conduct DPIA before high-risk processing activities

    Accountability (Article 5(2))

    Accountability Principle

    -- Detected patterns:
PROCESS personal_data WITHOUT audit_trail
HANDLE user_information missing documentation compliance_check

    Rule: COMP-GDPR-020 - Accountability Principle Impact: Violates GDPR Article 5(2) (accountability principle) Fix: Implement comprehensive audit trails and compliance monitoring

    Supervisory Authority Cooperation

    -- Detected patterns:
DENY supervisory_authority access audit investigation
REFUSE regulatory compliance request

    Rule: COMP-GDPR-018 - Supervisory Authority Cooperation Impact: Violates GDPR Article 31 (cooperation with authorities) Fix: Establish procedures for responding to authority requests

    üè• HIPAA Compliance Patterns (10 patterns)

    The Health Insurance Portability and Accountability Act (HIPAA) protects Protected Health Information (PHI). These detectors identify potential violations.

    Access Controls (45 CFR ¬ß164.308(a)(4))

    Insufficient PHI Access Controls

    -- Detected patterns:
SELECT patient_medical_data WHERE role!=physician
UPDATE health_record WHERE access_check=no validation

    Rule: COMP-HIPAA-001 - Insufficient PHI Access Controls Impact: Violates HIPAA 45 CFR ¬ß164.308(a)(4) (access control) Fix: Implement role-based access controls (RBAC) with proper authorization

    Audit Controls (45 CFR ¬ß164.312(b))

    Missing PHI Audit Trail

    -- Detected patterns:
ACCESS patient_information without_log
QUERY medical_data unlogged activity

    Rule: COMP-HIPAA-002 - Missing PHI Audit Trail Impact: Violates HIPAA 45 CFR ¬ß164.312(b) (audit controls) Fix: Implement comprehensive audit logging for all PHI access

    Integrity (45 CFR ¬ß164.312(c)(1))

    PHI Data Integrity Risk

    -- Detected patterns:
UPDATE medical_record without_backup
DELETE patient_data direct_table_access no integrity_check

    Rule: COMP-HIPAA-003 - PHI Data Integrity Risk Impact: Violates HIPAA 45 CFR ¬ß164.312(c)(1) (integrity) Fix: Implement data integrity controls including backups and checksums

    Authentication (45 CFR ¬ß164.312(d))

    Insufficient PHI Authentication

    -- Detected patterns:
CONNECT patient_database without_auth
LOGIN medical_system shared_credentials no password

    Rule: COMP-HIPAA-004 - Insufficient PHI Authentication Impact: Violates HIPAA 45 CFR ¬ß164.312(d) (person or entity authentication) Fix: Implement strong authentication including multi-factor authentication

    Minimum Necessary (45 CFR ¬ß164.502(b))

    Minimum Necessary PHI Access

    -- Detected patterns:
SELECT * patient_record WHERE need!=justified
ACCESS entire medical_chart unnecessary information

    Rule: COMP-HIPAA-005 - Minimum Necessary PHI Access Impact: Violates HIPAA 45 CFR ¬ß164.502(b) (minimum necessary) Fix: Limit PHI access to minimum necessary for intended purpose

    Transmission Security (45 CFR ¬ß164.312(e)(1))

    Unencrypted PHI Transmission

    -- Detected patterns:
SEND patient_data via HTTP unsecured
TRANSMIT medical_information plaintext unencrypted

    Rule: COMP-HIPAA-006 - Unencrypted PHI Transmission Impact: Violates HIPAA 45 CFR ¬ß164.312(e)(1) (transmission security) Fix: Implement TLS/SSL encryption for all PHI transmission

    Disposal (45 CFR ¬ß164.310(d)(2)(i))

    Improper PHI Disposal

    -- Detected patterns:
DELETE patient_data without_secure_deletion
DISPOSE medical_record no destruction_proof

    Rule: COMP-HIPAA-007 - Improper PHI Disposal Impact: Violates HIPAA 45 CFR ¬ß164.310(d)(2)(i) (disposal) Fix: Implement secure disposal procedures with proper destruction methods

    Breach Notification (45 CFR ¬ß164.404-408)

    HIPAA Breach Notification

    -- Detected patterns:
UNAUTHORIZED access patient_information suspicious activity
BREACH medical_data unusual export PHI

    Rule: COMP-HIPAA-008 - HIPAA Breach Notification Impact: Violates HIPAA 45 CFR ¬ß164.404-408 (breach notification) Fix: Implement breach detection, assessment, and notification procedures

    Business Associates (45 CFR ¬ß164.502(e))

    Business Associate Agreement

    -- Detected patterns:
SHARE patient_data with_third_party without_baa
PROVIDE medical_information to_vendor no agreement

    Rule: COMP-HIPAA-009 - Business Associate Agreement Impact: Violates HIPAA 45 CFR ¬ß164.502(e) (business associate agreements) Fix: Ensure all vendors handling PHI have signed BAAs

    Patient Rights (45 CFR ¬ß164.524, 164.526)

    Patient Rights Violation

    -- Detected patterns:
DENY patient access medical_record
REFUSE individual amend PHI request

    Rule: COMP-HIPAA-010 - Patient Rights Violation Impact: Violates HIPAA 45 CFR ¬ß164.524 (access to PHI) and ¬ß164.526 (amendment) Fix: Implement patient access and amendment procedures

    üí≥ PCI-DSS Compliance Patterns (5 patterns)

    The Payment Card Industry Data Security Standard (PCI-DSS) protects cardholder data. These detectors identify potential violations.

    Protect Stored Cardholder Data (Requirement 3)

    Insufficient Cardholder Data Protection

    -- Detected patterns:
SELECT credit_card_number unencrypted FROM transactions
STORE payment_information plaintext CVV

    Rule: COMP-PCI-001 - Insufficient Cardholder Data Protection Impact: Violates PCI DSS Requirement 3 (protect stored cardholder data) Fix: Implement strong encryption for all cardholder data storage

    Encrypt Transmission (Requirement 4)

    Unencrypted Cardholder Data Transmission

    -- Detected patterns:
SEND payment_data via HTTP unsecured
TRANSMIT credit_card_information plaintext

    Rule: COMP-PCI-002 - Unencrypted Cardholder Data Transmission Impact: Violates PCI DSS Requirement 4 (encrypt transmission) Fix: Implement TLS/SSL encryption for all transmissions

    Restrict Access (Requirement 7)

    Insufficient Cardholder Data Access Controls

    -- Detected patterns:
SELECT payment_transaction WHERE role!=authorized
ACCESS cardholder_data no_access_control

    Rule: COMP-PCI-003 - Insufficient Cardholder Data Access Controls Impact: Violates PCI DSS Requirement 7 (restrict access) Fix: Implement role-based access controls and limit access to necessary personnel

    Track and Monitor (Requirement 10)

    Missing Cardholder Data Audit Trail

    -- Detected patterns:
ACCESS payment_data without_log
QUERY cardholder_information no_audit_trail

    Rule: COMP-PCI-004 - Missing Cardholder Data Audit Trail Impact: Violates PCI DSS Requirement 10 (track and monitor access) Fix: Implement comprehensive audit logging for all cardholder data access

    Protect Data Retention (Requirement 3.1)

    Excessive Cardholder Data Retention

    -- Detected patterns:
RETAIN payment_data INDEFINITELY
STORE credit_card_information forever excessive retention

    Rule: COMP-PCI-005 - Excessive Cardholder Data Retention Impact: Violates PCI DSS Requirement 3 (protect stored data) Fix: Implement data retention policies and purge when no longer needed

    üõ°Ô∏è Compliance Best Practices

    1. Data Classification

    • Classify data by sensitivity level
    • Implement appropriate controls based on classification
    • Regular data inventory and mapping

    2. Access Controls

    • Principle of least privilege
    • Role-based access control (RBAC)
    • Regular access reviews and updates

    3. Audit and Monitoring

    • Comprehensive audit logging
    • Real-time monitoring and alerting
    • Regular compliance assessments

    4. Data Protection

    • Encryption at rest and in transit
    • Secure data disposal procedures
    • Data minimization practices

    5. Incident Response

    • Breach detection and response procedures
    • Regulatory notification processes
    • Documentation and reporting
